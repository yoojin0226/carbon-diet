# 🚀 통합 PRD: 우리 학교 기후환경 매니저

## 1. 프로젝트 개요 (Project Overview)

- **서비스명**: 우리 학교 기후환경 매니저
- **목표 사용자**: 용인시 수지구 관내 16개 중학교 학생 (추후 초등 고학년 및 고등학교 환경동아리로 확장).
- **핵심 가치**: 개인의 단순 기후행동 인증이 아닌, **학생들이 학교 공간의 에너지/환경 문제를 직접 관찰하고 제안하며 이를 취약계층 지원으로 연결**하는 진단 및 시민교육 플랫폼.
- **개발 환경 및 기술 스택**:
  - **프론트엔드**: Next.js, React, TypeScript
  - **백엔드/DB**: Supabase (또는 Mock Data 활용)
  - **배포**: Vercel
  - **디자인/코딩 보조**: Cursor, v0(브이제로), Stitch(구글 디자인 AI)
  - **버전 관리**: GitHub (기존 `school-diet` 레포지토리 활용 또는 신규 생성).

## 2. 핵심 개발 정책 및 제약 사항 (Constraints & Policies)

- **디바이스 최적화**: 학교 내 스마트폰 사용 불가 환경을 고려하여 **가로형 교육용 태블릿 PC 비율**을 최우선 기준으로 UI/UX를 설계합니다.
- **데이터 갱신 주기**: 경기기후플랫폼의 단위 면적당 탄소배출량 데이터는 매년 6월(1년 치)에 한 번 갱신됩니다. 따라서 월별 실시간 랭킹 시스템은 개발하지 않으며, '연간 학교 기후환경 리포트' 형태로 제공합니다.
- **AI 비용 절감(사진 인증 배제)**: AI 이미지 판독 기술은 토큰 사용 비용(운영비) 부담이 크므로 개발 범위에서 완전히 제외합니다. 대신 텍스트 기반의 체크리스트와 지도 핀(Pin) 기능을 사용합니다.
- **Mock Data 기반 우선 개발**: 실제 데이터 연동이 불가능할 경우를 대비하여, AI를 활용해 생성한 가짜(Mock) 데이터로 플랫폼을 우선 구성합니다.

## 3. 화면 및 기능 명세서 (MVP 10대 핵심 기능)

### [A] 온보딩 화면 (Onboarding)

사용자가 처음 접속했을 때 서비스의 핵심 흐름을 안내합니다.

1. **슬라이드 1 (역할 부여)**: "안녕? 난 우리 학교 기후환경 매니저 조아용이야!" 문구와 탐정 조아용 일러스트 노출.
2. **슬라이드 2 (관찰/기록)**: "너무 덥거나 전기가 낭비되는 곳을 지도에 콕 찝어줘!" 문구와 핀이 꽂히는 지도 이미지 노출.
3. **슬라이드 3 (개선/투표)**: "불편한 점을 찾았다면, 직접 해결 방법도 제안해 볼까요?" 문구와 투표함 상호작용 이미지 노출.
4. **슬라이드 4 (보상/기부)**: "모은 포인트로 조아용을 꾸미고, 이웃에게 따뜻함을 선물해요." 문구와 기부 매칭 일러스트 노출 후 `[시작하기]` 버튼 제공.

### [B] 메인 대시보드 (Main Dashboard)

학생이 로그인 후 가장 먼저 보는 화면으로 6가지 핵심 요소가 포함됩니다.

1. **조아용 꾸미기 (단기 보상)**: 학생이 상점에서 구매한 모자, 망토 등으로 커스텀된 조아용 캐릭터를 렌더링합니다.
2. **우리 학교 기후환경 레벨**: 학교 단위의 총 활동량(관찰 및 제안 수)에 따라 나무/숲이 성장하는 UI 프로그레스 바를 제공합니다.
3. **오늘의 매니저 미션**: "빈 교실 전등 끄기", "더운/추운 곳 찾기" 등 계절에 맞춘 텍스트 기반 일일 관찰 퀘스트 리스트를 제공합니다.
4. **이번 달 개선 제안 TOP 3**: `[C]`에서 공감(투표)을 가장 많이 받은 제안 3개의 제목과 상태(검토 중, 반영 완료 등)를 카드 형태로 렌더링합니다.
5. **내 포인트 요약**: 현재 보유 포인트 잔액을 보여주며, 클릭 시 '상점'이나 '기부' 페이지로 라우팅됩니다.
6. **우리 학교 에너지 카드 (데이터 학습)**: 경기기후플랫폼 데이터를 변환하여 "우리 학교 공간 1칸을 유지하는 데 나오는 탄소의 양" 형태의 교육용 텍스트 및 카드뉴스를 슬라이더로 제공합니다.

### [C] 관찰 및 제안 공간 (Observation & Proposal)

1. **우리 학교 기후환경 지도 (관찰 기능)**:
  - 학교 평면도 UI를 띄우고 사용자가 특정 위치를 터치하면 핀(Pin)을 생성합니다.
  - 핀 종류 선택: 더운 공간, 추운 공간, 전기 낭비(빈 교실 전등), 물 낭비, 시설 불편.
  - *개발 참고*: 위치별 좌표 데이터(X, Y)와 핀 카테고리 Enum을 DB에 저장해야 합니다.
2. **구조화된 환경 개선 제안함 (제안 기능)**:
  - 학생이 자유 텍스트가 아닌 **6가지 고정 질문**(어디에서 발견했나요?, 어떤 문제가 있나요?, 왜 문제인가요?, 어떻게 바꾸면 좋을까요?, 누구에게 도움이 될까요?, 에너지와 어떤 관련이 있나요?)에 답을 입력하는 Form을 제출하도록 구현합니다.
3. **제안 공감 및 투표 (상호작용 기능)**:
  - 게시된 제안에 대해 `[공감하기]` 버튼(카운트 +1)을 제공하며, 댓글 형태로 보완 의견("이렇게 하면 더 좋을 것 같아요")을 작성하는 기능을 추가합니다.

### [D] 보상 및 기부 공간 (Reward & Donation)

1. **조아용 상점**: 보유 포인트를 차감하여 캐릭터 파츠(모자, 옷 등)를 DB에 `purchased_items` 배열로 업데이트하고 메인 렌더링에 반영합니다.
2. **포인트 기부**: "에너지 취약계층 냉/난방 지원" 버튼을 클릭해 포인트를 차감하고, 이를 누적 기부 게이지(학교 단위 통합 기부금)에 합산하여 렌더링합니다.

## 4. 데이터베이스 및 포인트 정책 (Developer Guide)

개발자가 DB 스키마 및 비즈니스 로직을 구성할 때 사용할 핵심 수치입니다.

### 4-1. 포인트 획득 로직 (Point System)

- **관찰 기록**: 교실 쾌적도 체크(3pt), 특정 공간 불편/낭비 신고(5pt). *단, 어뷰징 방지를 위해 동일 위치/동일 문제는 쿨타임(중복 방지 로직) 적용 필수.*
- **개선 제안**: 구조화된 폼 작성 완료(10pt), 사진/그림 첨부 시(+3pt).
- **상호 작용**: 타인의 제안에 공감 투표(1pt / 일 최대 3회 제한), 보완 의견 작성(3pt).
- **데이터 학습**: 에너지 카드 읽기(1pt), 관련 퀴즈 풀기(1pt).

### 4-2. DB 테이블 구조화 팁 (Supabase 기준)

- `Users`: 학생 ID, 소속 학교명, 학급, 보유 포인트, 구매한 코스튬 ID 배열.
- `Observations` (관찰 지도용): 학교 ID, X좌표, Y좌표, 카테고리(더움/추움/낭비 등), 작성자 ID, 생성 일자.
- `Proposals` (구조화된 제안함): 제안 ID, 학교 ID, 작성자 ID, q1_where, q2_what, q3_why, q4_how, q5_who, q6_energy_relation, 공감 수(like_count), 진행 상태(status).
- `Point_Logs` (포인트 관리): 유저 ID, 변동량(+/-), 변동 사유(관찰, 제안, 기부, 상점 구매), 발생 일자.
- `School_Data` (Mock 데이터용): 학교 ID, 올해 단위면적당 탄소배출량, 작년 배출량, 레벨(경험치).

**진행 제안**: 개발 초기에는 Next.js 환경 설정과 Vercel 배포를 가장 먼저 테스트 하신 후, 위 기능 명세 중 `[B] 메인 대시보드`의 6가지 요소를 정적 화면으로 구성하는 것부터 시작하시기를 추천합니다.



## 상세 디자인 가이드

## **1) 홈 탭 상세(1번째 이미지) 분석**

### **레이아웃/그리드**

- **전체 배경**: 상단은 거의 흰색, 하단으로 갈수록 **아주 연한 하늘색 그라데이션**(화이트 → 라이트 블루).
- **상단 상태바 영역**
  - **좌측**: 물방울 아이콘 + 숫자(재화/포인트) `1,250`
  - **우측**: 작은 아이콘 2개(예: 인박스/알림 느낌) — 우측 여백을 충분히 둔 미니멀 배치
- **타이틀 영역(상단 중앙 정렬)**
  - 텍스트: `우리 학교 순위 15위 째`
  - 가운데 정렬, 상단 여백 큼(‘공기감’ 크게)
  - 숫자 `15`만 **초록색 강조**, 나머지는 다크 그레이/블랙
- **메인 비주얼(중앙)**
  - 캐릭터 3D/일러스트 오브젝트가 **가운데 하단 쪽**에 “떠있는” 느낌으로 배치
  - 화면 여백이 많고, 캐릭터가 작은 편(‘정적·힐링’ 톤)
- **하단 콘텐츠 카드(퀴즈 카드)**
  - 하단 탭바 바로 위에 **큰 가로형 카드 1장**
  - 카드 내부:
    - 상단에 작은 라벨 텍스트(초록색) `오늘의 기후 퀴즈`
    - 아래에 질문 텍스트(검정) 큰따옴표 포함
- **하단 탭바(고정)**
  - 5개 탭 아이콘 + 라벨
  - 선택된 탭만 **초록색**, 비선택 탭은 그레이
  - 탭바 배경은 흰색에 가까운 단색, 상단 경계가 매우 약함(그림자 거의 없음)

### **컬러(느낌/팔레트)**

- **포인트 컬러(메인)**: 연두~초록 계열(선택 탭, 강조 숫자, 카드 라벨)
- **서브 컬러**: 하늘색(배경 그라데이션), 아주 연한 민트 느낌
- **텍스트**
  - 제목/핵심: 거의 블랙(#111~#222 느낌)
  - 보조: 미디엄 그레이(#8A~#A0 느낌)
- **카드**
  - 배경: 화이트
  - 테두리: 거의 없는 수준(있더라도 아주 옅은 회색)
  - 그림자: 매우 소프트(“뜬” 느낌만 살짝)

### **컴포넌트 스타일 가이드(재현용)**

- **Top 재화 표시**
  - 아이콘(물방울) + 숫자: 좌측 정렬, 숫자는 **세미볼드**
- **순위 텍스트**
  - 한 줄 텍스트 중 **특정 토큰(숫자)만 색상 강조**하는 “부분 강조” 컴포넌트
- **퀴즈 카드**
  - 라운드: 큼(대략 18~24px 이상 느낌)
  - 패딩: 상하 충분(특히 좌우가 넓음)
  - 라벨: 작은 폰트 + 초록색
  - 본문: 한 줄/두 줄 정도, 가운데 정렬에 가까움(카드 중앙에 안정적으로 놓임)
- **탭바**
  - 아이콘은 컬러가 핵심(활성=초록, 비활성=회색)
  - 라벨은 작은 크기, 아이콘 아래 1줄

## **2) 상점 내 ‘인테리어 테마’ 카드 그리드(2번째 이미지) 분석**

### **레이아웃/그리드**

- **카드 그리드**: 2행 × 3열로 보이는 구성(6개 카드)
- 각 카드 구성:
  - **좌상단 배지**: `D-5` 같은 빨간 배지(둥근 사각형/필 형태)
  - **상단 일러스트 영역**: 카드 상단의 큰 그림(파스텔 일러스트)
  - **타이틀 텍스트**: 카드 중앙 하단쯤에 굵게(예: `나의 작은 구름 위`, `친환경 자전거 여행` 등)
  - **진행 표시**: 최하단에 `0/12` 형태(큰 숫자, 중앙 정렬)
- 카드 배경은 화이트, 카드들 사이 간격이 일정하고 여백이 깔끔함

### **컬러/톤**

- **일러스트 중심 파스텔 팔레트**(연보라/연핑크/연하늘/연노랑)
- **배지(D-5)**: 강한 레드(주의/기간 강조)
- 텍스트는 블랙/다크그레이, 진행표시는 진한 회색

### **컴포넌트 규칙**

- **상단 배지**
  - 위치: 카드 좌상단 “붙임”
  - 형태: 둥근 캡슐(필) / 배경 레드 / 텍스트 화이트
- **카드 라운드**
  - 라운드 큼(16~22px 느낌)
  - 그림자 거의 없음, 대신 아주 옅은 외곽/톤 차로 카드 구분
- **진행 표시(0/12)**
  - 별도 바 없이 **숫자만 큼직**하게 중앙 배치(진행이 직관적)

## **3) 홈 탭 안 ‘상점 상세’(3번째 이미지) 분석**

### **레이아웃(상단 → 하단)**

- **상단 재화 표시**: 좌측 상단 물방울 + `1,250`
- **상단 카테고리 탭(텍스트 탭)**
  - `꾸미기 / 옷장 / 충전소` 3탭
  - 활성 탭(꾸미기) 아래에 **초록색 얇은 언더라인**
  - 비활성 탭은 회색
- **아이템 목록(카드 리스트/그리드)**
  - 2행 × 3열 형태로 보이는 **라운드 카드**들
  - 카드 내부: 아이콘(작은 그림) + 아이템명 + 가격(물방울 아이콘 + 숫자)
  - 카드 배경은 흰색, 외곽 라운드 큼, 구분선/그림자 최소
- **섹션 타이틀**
  - `인테리어 테마` 같은 섹션 헤더는 좌측 정렬, 굵게
- **인테리어 테마 카드(대형 2열)**
  - 큰 카드 2개가 가로로 배치(좌/우)
  - 카드 내부는 “큰 이미지 프레임” 중심
  - 카드 하단에는:
    - 테마명(굵게)
    - 진행표시(예: `0/10`, `0/6`)가 **회색 캡슐 필** 안에 들어감
- **하단 탭바(고정)**
  - 5탭: `홈 / 퀘스트 / 랭킹 / 스토어 / 내 정보`
  - 활성 탭은 초록색(여기선 `스토어`)
  - 비활성은 회색

### **컬러/타이포**

- **메인 강조색**: 초록(탭 언더라인, 하단 탭 활성)
- **가격 표시**: 물방울 아이콘 + 숫자(파란 계열로 보이기도 함)로 “재화” 일관성 유지
- **배경**: 거의 화이트(상단 탭 영역도 아주 밝음)
- **카드**: 흰 바탕 + 아주 약한 구분선/톤 차

### **컴포넌트 상세(버튼/카드/배치 규칙)**

- **텍스트 탭(상단)**
  - 탭 간 간격 넓음, 중앙 정렬
  - 활성 상태는 “초록 언더라인 + 텍스트 색 조금 진해짐”
- **아이템 카드(소형)**
  - 내부 구성: (작은 아이콘) → (아이템명) → (가격)
  - 가격은 “재화 아이콘 + 숫자”를 항상 함께 표시
  - 카드 높이/폭이 균일(그리드 정렬이 핵심)
- **인테리어 테마 카드(대형)**
  - 카드 자체는 라운드 큼, 내부는 “큰 이미지 캔버스(여백 많은 프레임)” 느낌
  - **진행 캡슐(0/10)**:
    - 배경: 연회색
    - 라운드: 완전 필
    - 텍스트: 진한 회색
- **하단 탭바**
  - 아이콘+라벨 세로 스택
  - 활성색만 변경(배경은 유지), 전체가 매우 깔끔한 “플랫” 톤

## **요약: 이 UI를 그대로 구현하려면(핵심 규칙만)**

- **여백을 크게 쓰는 ‘힐링형’ 레이아웃**(상단/중앙 공간 넓게)
- **화이트 + 파스텔 그라데이션 + 초록 포인트**(활성/강조는 초록으로 통일)
- **카드 라운드 크게 + 그림자 최소**(선/톤 차로 구분)
- **진행/카운트는 숫자만으로도 크게 표현**(0/12, 0/10 같은 형태)
- **탭 UI 2종**
  - 상단 카테고리 탭: 텍스트 + 언더라인
  - 하단 네비 탭: 아이콘 + 라벨, 활성색만 변경
